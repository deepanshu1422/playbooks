  • Spending 60 min on Hard problem, missing Easy ones

Round 2: Technical Interview (DSA Focus)

Format:

  • Duration: 45-60 minutes
  • 1-2 coding problems (Medium, sometimes Hard)
  • Live coding (Google Doc, CoderPad, or IDE share)
  • Interviewer watches you code

Strategy:

Introduction (2 min):

  • Greet warmly, smile (even if virtual)
  • "Hi, I'm [Name], final year CSE at Chitkara. I enjoy problem-solving and building web apps."
  • Keep it crisp (they want to get to coding fast)

Problem Solving (REACTO framework - 40 min):

  • Listen carefully: Take notes as they explain
  • Clarify: "Just to confirm, the array can have negative numbers?"
  • Example: "Let me trace through: if input is [1,2,3], output should be..."
  • Brute Force First: "The simplest approach would be O(n¬≤)... but we can optimize."
  • Think Aloud: "I'm thinking a hash map could help here because..."
  • Code Slowly: Better to write correct slow code than fast buggy code
  • Test: "Let me trace through my code with the example..."
  • Fix Bugs: If you spot error, don't panic - "Oh, I see the issue, let me fix this..."

Wrap-Up (3 min):

  • "I think this solution is optimal. Can we discuss any further optimizations?"
  • If time left: Be ready for follow-up (modify problem, discuss complexity)

Communication Tips:

  • Think aloud: Silence = interviewer doesn't know if you're thinking or stuck
  • Ask for hints: If stuck for 5 min, say "Could you give me a hint on the approach?"
  • Admit unknowns: "I haven't seen this exact problem, but I'll try approaching it as..."
  • Stay calm: Nervousness is normal, but don't let it paralyze you

Common Mistakes to Avoid:

  • Jumping to code without discussing approach (they stop you mid-way)
  • Not testing code (leads to bugs)
  • Defensive behavior when hint given ("No, I got it" - just listen!)
  • Giving up ("I don't know" after 2 min - keep trying!)

Format:

  • First 15 min: Project discussion
  • Next 30 min: System design OR another DSA problem
  • Last 15 min: Core CS questions

Strategy:

  • Structure: "I built [X], which solves [problem]. Tech stack was [Y]. Key challenge was [Z]."
  • Metrics: "The app handles 1000 products and 50 concurrent users."

  • Depth: They test how deep you understand (be ready for architectural questions)

Example Deep Dive:

  • Interviewer: "How does authentication work in your app?"
  • You: "I use JWT. On login, server generates a token with user ID and expiry, signed with a secret. Client stores it in localStorage and sends in Authorization header for protected routes. Server verifies signature and checks expiry."
  • Interviewer: "What if token is stolen?"
  • You: "Good question. I'd implement refresh tokens (short-lived access token, longer refresh token stored in HTTP-only cookie for security)."

System Design (Basics):

  • Question: "Design a URL shortener."
  • Approach:
    1. Requirements: "Functional: Shorten URL, redirect. Non-functional: Low latency, high availability, scalable to 100M URLs."
    2. API: POST /shorten (long URL ‚Üí short URL), GET /{shortURL} (redirect)
    3. Database: SQL (low scale) or NoSQL (high scale). Schema: shortURL (key), longURL, createdAt, expiryAt
    4. Hashing: Use Base62 encoding (62 chars: a-z, A-Z, 0-9) for short URL. 6 chars = 62^6 = 56B combinations.
    5. Scaling: Add caching (Redis for popular URLs), load balancer, database sharding (by hash of short URL).
    6. Trade-offs: Consistency vs Availability (CAP theorem - prioritize Availability for our use case).

This is normal. Most students face this.

  • Course-correct: If aptitude failed, do 50 problems today. If coding failed, solve 20 easy problems fast.
  • Mock interviews: Do 2 mocks daily to build confidence
  • Mental health: Talk to peers/seniors (you're not alone), take 1 day off if burnout
  • Widen net: Apply to companies you initially skipped

Example: You have TCS offer (‚Çπ4 LPA, accept by Friday) but Nagarro drive (‚Çπ9 LPA) is on Saturday.

  2. Request extension: Email TCS HR explaining situation (low success rate, but worth trying).
  3. Accept TCS, skip Nagarro: Safe, but opportunity cost high.

  • Startups: AngelList, YC companies hiring freshers
  • Referrals: Ask every senior/alumni for referral
  • Backup: Consider service-based companies with rolling hiring (TCS, Infosys hire year-round)
  • Upskilling: If market is tough, do 3-month internship (can convert to full-time)

  • After each rejection, write: What went wrong? What will I do differently?
  • Review before next drive (avoid repeating mistakes)
  • Psychological benefit: Converts failure to learning

Tip 4: The "Interviewer Mindset"

  • When solving problems, think: "Would I hire myself?"
  • Code quality matters: Clean variable names, comments, structure
  • Communication: Explain clearly, admit unknowns confidently
  • Attitude: Curious, collaborative, not defensive

Tip 5: The "One Hour Rule"

  • If stuck on a problem for 1 hour, stop. Read solution.
  • Understand why it works, implement from scratch (no copy-paste)
  • Move on. (1 understood solution > 3 hours of frustration)

Tip 6: The "Teach to Learn"

  • Explain concepts to peers (Feynman technique)
  • If you can't explain simply, you don't understand deeply
  • Bonus: Helps peers, builds network

Tip 7: The "Glassdoor Goldmine"

  • Before every company drive, read 10 recent interview experiences
  • Note common patterns (e.g., "Nagarro always asks string problems")
  • Prepare accordingly (company-specific edge)

Tip 8: The "Energy Management"

  • Placements are a marathon, not sprint
  • Schedule rest: 1 day off per week (no coding, no prep - just recharge)
  • Exercise: 30 min daily (walk, yoga, gym - reduces anxiety)
  • Sleep: Non-negotiable 7 hours (tired brain = failed interviews)

Tip 9: The "Network Effect"

  • Connect with seniors who got placed in your target companies
  • Ask for insider tips (interview pattern, culture, dos/don'ts)
  • Request referrals (if they're willing, post you get offer - for next switch)

Tip 10: The "Gratitude Practice"

  • Daily: Write 3 things you're grateful for (reduces stress)
  • Placement season is intense; mental health is critical
  • Celebrate small wins (shortlisted for next round? Treat yourself!)
